## 核对标准

每个音标必须满足：

1. 画面中正在讲该音标，或该音标被高亮/展示。
2. 音频中主持人正在介绍或带读该音标。
3. `start` 应包含主持人必要铺垫，允许提前 1-2 秒。
4. `end` 到下一个音标正式开始前，不能截断讲解。
5. 对不确定的条目标记 `confidence: "low"`，不得伪装成已确认。

## JSON 建议格式

```json
[
  {
    "index": 1,
    "symbol": "/i:/",
    "start": 79.0,
    "end": 118.0,
    "confidence": "verified",
    "evidence": "画面显示 /i:/，主持人开始讲长元音 /i:/"
  }
]
```
